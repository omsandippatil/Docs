Introduction

Welcome to the Full-Stack Social Media Application project! This application is built using Next.js 15, React Query, Lucia Auth, TypeScript, and Tailwind CSS, combining modern web development tools to create a fully functional social media platform. The project includes features such as user authentication, real-time data fetching, infinite scrolling, notifications, and much more.

Whether you're looking to build a scalable, feature-rich application or learn advanced techniques in web development, this project will take you through the entire process step-by-step. From setting up the backend database using Prisma and Postgres to deploying a production-ready app, this tutorial is designed to guide you through every aspect of the development process.

With real-time messaging, file uploads, pagination, and responsive design, this project simulates the complex functionality of real-world social media apps, making it an ideal learning experience for developers looking to sharpen their full-stack skills.

Let’s dive in and build something amazing!

Technologies & Components

  1. Next.js 15:

    • Framework for building server-rendered React applications.
    • Features like server components, server actions, and a new app router for advanced data handling.
    • Supports responsive layouts, enabling the app to work on both mobile and desktop.

  2. React Query:

    • Library for data fetching, caching, and optimistic updates.
    • Allows for infinite loading and stale data revalidation.
    • Optimizes the user experience by immediately updating UI after actions like liking a post, even before backend confirmation.

  3. Lucia Auth:

    • Authentication system replacing NextAuth, providing more flexibility.
    • Supports various authentication methods such as username/password and Google Sign-In.
    • Paired with React Hook Form for real-time form validation.

  4. Tailwind CSS:

    • Utility-first CSS framework for styling.
    • Customizable with CSS variables, and responsive design built-in.

  5. ShadCN/UI:

    • Component library built on Radix UI.
    • Provides flexible, customizable components like buttons, forms, modals, and notifications.
    • Allows copying components directly into the project for full control and customization.

  6. Prisma ORM:

    • ORM (Object-Relational Mapper) for interacting with a Postgres database.
    • Defines the database schema and enables querying with predefined functions instead of raw SQL.
    • Allows for full-text search and data caching.

  7. Postgres:

    • SQL-based database hosted on platforms like Neon or Vercel.
    • Used as the main data storage for user data, posts, comments, etc.

  8. UploadThing:

    • Wrapper for Amazon S3 for file uploads.
    • Supports drag-and-drop functionality and the upload of images/videos for posts.

  9. Stream Chat SDK:

    • Provides real-time messaging and file uploads for direct messaging between users.
    • Integrated for building a fully functional chat system with minimal configuration.

  10. ESLint, Prettier, and Tailwind IntelliSense:

    • Tools used for code formatting, linting, and intelligent autocomplete for Tailwind classes.
    • Enhances the developer experience by maintaining code quality and consistency.

  11. Cron Jobs:

    • Used for periodic maintenance tasks like deleting unused files from the database to prevent bloating.

  • Dark/Light Themes: Switchable themes, adapting to system defaults.

1. Authentication using Lucia Auth

Lucia Auth provides flexible authentication, allowing for both email/password and Google sign-in. Below is a snippet of how authentication is handled:

// lucia.ts
import { lucia } from "lucia";
import { nextjs } from "lucia-auth/nextjs";

// Initialize Lucia with Next.js
const auth = lucia({
  adapter: nextjs(),
  env: process.env.NODE_ENV === "production" ? "PROD" : "DEV",
  transformUserData: (user) => {
    return {
      userId: user.id,
      username: user.username,
    };
  },
});

export default auth;

Explanation:

  • lucia is initialized with a Next.js adapter.
  • The transformUserData function extracts necessary user data, which is later used for authentication.
  • This setup supports both email/password and Google sign-in.

2. React Query for Data Fetching and Caching

React Query is used for efficient data fetching with features like infinite scrolling, optimistic updates, and caching.

// usePosts.ts
import { useQuery, useMutation, useQueryClient } from 'react-query';
import axios from 'axios';

// Fetch posts with pagination
export const usePosts = (page = 1) => {
  return useQuery(['posts', page], async () => {
    const { data } = await axios.get(`/api/posts?page=${page}`);
    return data;
  });
};

// Mutate post (like/unlike)
export const useToggleLike = () => {
  const queryClient = useQueryClient();

  return useMutation(
    async (postId: string) => {
      return axios.post(`/api/posts/${postId}/toggle-like`);
    },
    {
      // Optimistically update the cache
      onMutate: async (postId) => {
        await queryClient.cancelQueries('posts');
        const previousPosts = queryClient.getQueryData('posts');
        queryClient.setQueryData('posts', (old: any) => {
          return old.map((post: any) => 
            post.id === postId ? { ...post, liked: !post.liked } : post
          );
        });
        return { previousPosts };
      },
      onError: (err, postId, context: any) => {
        queryClient.setQueryData('posts', context.previousPosts);
      },
      onSettled: () => {
        queryClient.invalidateQueries('posts');
      },
    }
  );
};

Explanation:

  • useQuery is used to fetch posts from the server, paginated by page.
  • useMutation is used to toggle likes on posts with optimistic updates.
  • The UI reflects changes instantly before the server confirms, thanks to optimistic updates.

3. File Uploading with UploadThing and Amazon S3

File uploading is managed via UploadThing which wraps Amazon S3 for storage. The app supports drag-and-drop, copy-paste, and multiple file uploads.

// upload.ts
import { useState } from 'react';
import { UploadThing } from 'uploadthing/react';

export default function UploadComponent() {
  const [files, setFiles] = useState([]);

  const handleUpload = async (fileList: FileList) => {
    const response = await UploadThing.upload({
      files: fileList,
    });
    setFiles([...files, ...response]);
  };

  return (
    <div>
      <input
        type="file"
        multiple
        onChange={(e) => handleUpload(e.target.files)}
      />
      <div>
        {files.map((file) => (
          <img key={file.url} src={file.url} alt="Uploaded file" />
        ))}
      </div>
    </div>
  );
}

Explanation:

  • File uploads are handled by UploadThing, which interacts with Amazon S3.
  • Files are uploaded by selecting or dragging them into the input.
  • The uploaded files are displayed after successful upload, with their URLs rendered in the UI.

4. Real-time Chat with Stream Chat SDK

Real-time chat functionality uses the Stream Chat SDK for features like messaging, file attachments, and reactions.

// chat.ts
import { useEffect, useState } from 'react';
import { StreamChat } from 'stream-chat';

const chatClient = StreamChat.getInstance('your_api_key');

export default function Chat() {
  const [messages, setMessages] = useState([]);
  
  useEffect(() => {
    const channel = chatClient.channel('messaging', 'general', {
      name: 'General Chat',
    });

    const initChat = async () => {
      await channel.watch();
      channel.on('message.new', (event) => {
        setMessages((prev) => [...prev, event.message]);
      });
    };

    initChat();
  }, []);

  const sendMessage = async (text) => {
    await channel.sendMessage({ text });
  };

  return (
    <div>
      {messages.map((msg, index) => (
        <p key={index}>{msg.text}</p>
      ))}
      <input type="text" onKeyDown={(e) => e.key === 'Enter' && sendMessage(e.target.value)} />
    </div>
  );
}

Explanation:

  • A Stream Chat client is created and connects to the general channel.
  • New messages are fetched and rendered in real-time, with the UI updating whenever a new message is sent or received.
  • Users can send messages using the input field.

5. Database Integration with Prisma

The project uses Prisma ORM to interact with a Postgres database for managing users, posts, and comments.

// schema.prisma
model User {
  id        String   @id @default(uuid())
  username  String   @unique
  email     String   @unique
  posts     Post[]
}

model Post {
  id        String   @id @default(uuid())
  title     String
  content   String
  author    User     @relation(fields: [authorId], references: [id])
  authorId  String
}

// prismaClient.ts
import { PrismaClient } from '@prisma/client';

const prisma = new PrismaClient();

export default prisma;

Explanation:

  • The Prisma schema defines models for User and Post, establishing relationships between users and their posts.
  • prismaClient.ts initializes Prisma to allow database interactions like creating users, querying posts, etc.

6. Real-time Notifications with React Query

Notifications are built using React Query with automatic revalidation for real-time updates.

// notifications.ts
import { useQuery } from 'react-query';
import axios from 'axios';

export const useNotifications = () => {
  return useQuery('notifications', async () => {
    const { data } = await axios.get('/api/notifications');
    return data;
  }, {
    refetchInterval: 60000,  // Refetch every minute
  });
};

Explanation:

  • Notifications are fetched via useQuery, and refetch every minute to keep the count updated.
  • The app displays notifications and resets the unread count after users view them.
